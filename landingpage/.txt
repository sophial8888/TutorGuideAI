Build the TutorGuide AI landing page as a React + Vite + TailwindCSS site (frontend) with a Flask API backend.

Product: AI-powered math tutoring chatbot for K-12 students, designed to support peer tutors.
Tone: "Fun but credible" — warm, energetic, and approachable while remaining trustworthy.

Typography:
- Fredoka for headings and Nunito for body text via CSS variables. Only use these two fonts.
- Interaction behavior: Hero words "Made" (blue) and "Smarter" (pink) use cursor-default but have hover scale effects.

CRITICAL FIDELITY CONSTRAINTS
- Color Palette: Blue (#5BC0EB), Yellow (#FFD93D), Pink (#FF8FAB), Green (#6EE7B7), Dark (#2B2B2B).
- Selection color: `selection:bg-[#5BC0EB] selection:text-white`.
- No smooth scrolling should be added beyond the `scroll-smooth` class on the html tag.
- The header must have a `backdrop-blur-md` and `bg-white/90` with a transition duration of 300ms.

TECH STACK / DEPENDENCIES
- Tailwind CSS (via CDN: https://cdn.tailwindcss.com)
- Iconify Icon Component (https://code.iconify.design/iconify-icon/1.0.7/iconify-icon.min.js)
- Google Fonts: Fredoka (400, 500, 600) and Nunito (400, 500, 600, 700).

GLOBAL STYLE
- Body: `font-family: 'Nunito', sans-serif; color: #2B2B2B; background-color: #FAFAFA;`.
- Headings: `font-family: 'Fredoka', sans-serif;`.
- Custom animations:
  - `float`: translateY(0) to translateY(-15px) over 6s infinite.
  - `wiggle`: rotate(-3deg) to rotate(3deg) over 0.5s infinite (triggered on hover).
  - `bounce`: standard bounce with 3s duration.

GLOBAL ANIMATION / INTERACTION RULES
- All hover transitions should be 300ms duration.
- Horizontal scroll areas must use `snap-center` on child elements.
- `no-scrollbar` utility must be used for testimonial containers.

COMMON MISTAKES TO AVOID
- Do not use standard squares; the site uses heavily rounded corners (`rounded-[2rem]` and `rounded-[3rem]`).
- Do not miss the `rotate-3` and `-rotate-2` background layers behind the hero image.
- Ensure the float animation is infinite and the wiggle is only on hover.

IMPLEMENTATION REQUIREMENTS
- Build exactly as described. Use the original asset URLs. Ensure responsive breakpoints (md: grid columns) match the layout descriptions.
